1. Introduction aux Arbres de Décision

🎯 Qu'est-ce qu'un arbre de décision ?

Un arbre de décision est un modèle d'apprentissage automatique utilisé pour la classification et la régression. Il représente graphiquement une série de décisions sous forme d'arbre où :

  • Nœuds internes : tests sur les attributs (conditions)
  • Branches : résultats possibles des tests
  • Feuilles : décisions finales (classes ou valeurs)

Exemple simple : Décider de sortir ou non

                    Météo ?
                   /       \
               Soleil     Pluie
                 |          |
              SORTIR    Température ?
                         /       \
                      < 5°C     ≥ 5°C
                        |         |
                   RESTER    Vent fort ?
                             /       \
                          OUI        NON
                           |          |
                       RESTER      SORTIR

💡 Avantages

  • ✅ Interprétable : facile à comprendre et visualiser
  • ✅ Peu de prétraitement : pas besoin de normalisation
  • ✅ Multi-classes : gère plusieurs catégories
  • ✅ Variables mixtes : numériques et catégorielles

⚠️ Inconvénients

  • ❌ Surapprentissage : risque élevé sans élagage
  • ❌ Instabilité : sensible aux variations des données
  • ❌ Biais : favorise les attributs avec beaucoup de valeurs

2. Construction d'un Arbre de Décision

Algorithmes principaux

  • ID3 (Iterative Dichotomiser 3) : utilise le gain d'information basé sur l'entropie
  • C4.5 : amélioration d'ID3, gère les valeurs continues et manquantes
  • CART (Classification And Regression Trees) : utilise l'indice de Gini

📊 Entropie et Gain d'Information

1. Entropie (mesure de désordre)

H(S) = -Σ pᵢ · log₂(pᵢ)

où pᵢ est la proportion de la classe i dans l'ensemble S

Exemple de calcul d'entropie

Dataset : 9 exemples "Oui", 5 exemples "Non"

p(Oui) = 9/14, p(Non) = 5/14

H(S) = -(9/14)·log₂(9/14) - (5/14)·log₂(5/14)
H(S) ≈ 0.940

➜ Entropie proche de 1 = beaucoup de désordre (mélange équilibré)

➜ Entropie proche de 0 = peu de désordre (classe dominante)

2. Gain d'Information

Gain(S, A) = H(S) - Σ (|Sᵥ| / |S|) · H(Sᵥ)

où A est un attribut, Sᵥ sont les sous-ensembles créés par les valeurs v de A

🎯 Principe : On choisit l'attribut qui maximise le gain d'information (qui réduit le plus l'entropie).

📐 Indice de Gini (utilisé par CART)

Gini(S) = 1 - Σ pᵢ²

Exemple de calcul de Gini

Dataset : 9 "Oui", 5 "Non"

Gini(S) = 1 - [(9/14)² + (5/14)²]
Gini(S) ≈ 0.459

➜ Gini = 0 → pureté totale (une seule classe)

➜ Gini = 0.5 → impureté maximale (classes équilibrées pour 2 classes)

4. Implémentation en Python

Code simple avec sklearn

from sklearn.tree import DecisionTreeClassifier
from sklearn.preprocessing import LabelEncoder
import pandas as pd

# Données d'exemple
data = {
    'Âge': ['Jeune', 'Jeune', 'Moyen', 'Senior', 'Senior', 'Senior',
            'Moyen', 'Jeune', 'Jeune', 'Senior', 'Jeune', 'Moyen',
            'Moyen', 'Senior'],
    'Revenus': ['Élevé', 'Élevé', 'Élevé', 'Moyen', 'Faible', 'Faible',
                'Faible', 'Moyen', 'Faible', 'Moyen', 'Moyen', 'Moyen',
                'Élevé', 'Moyen'],
    'Étudiant': ['Non', 'Non', 'Non', 'Non', 'Oui', 'Oui',
                 'Oui', 'Non', 'Oui', 'Oui', 'Oui', 'Non',
                 'Oui', 'Non'],
    'Crédit': ['Bon', 'Excellent', 'Bon', 'Bon', 'Bon', 'Excellent',
               'Excellent', 'Bon', 'Bon', 'Bon', 'Excellent', 'Excellent',
               'Bon', 'Excellent'],
    'Achète': ['Non', 'Non', 'Oui', 'Oui', 'Oui', 'Non',
               'Oui', 'Non', 'Oui', 'Oui', 'Oui', 'Oui',
               'Oui', 'Non']
}

df = pd.DataFrame(data)

# Encoder les variables catégorielles
le = LabelEncoder()
for col in df.columns:
    df[col] = le.fit_transform(df[col])

# Séparer features et target
X = df.drop('Achète', axis=1)
y = df['Achète']

# Créer et entraîner l'arbre
clf = DecisionTreeClassifier(criterion='entropy', max_depth=3)
clf.fit(X, y)

# Prédiction
nouveau = [[1, 1, 0, 0]]  # Moyen, Élevé, Non, Bon
prediction = clf.predict(nouveau)
print(f"Prédiction : {'Oui' if prediction[0] == 1 else 'Non'}")

# Importance des features
for i, importance in enumerate(clf.feature_importances_):
    print(f"{X.columns[i]}: {importance:.3f}")

Implémentation from scratch

import numpy as np
from collections import Counter

class NoeudDecision:
    def __init__(self, feature=None, seuil=None, gauche=None, droite=None, valeur=None):
        self.feature = feature      # Index de l'attribut à tester
        self.seuil = seuil          # Valeur seuil pour la division
        self.gauche = gauche        # Sous-arbre gauche
        self.droite = droite        # Sous-arbre droite
        self.valeur = valeur        # Valeur de prédiction (feuille)

class ArbreDecision:
    def __init__(self, max_profondeur=10, min_echantillons=2):
        self.max_profondeur = max_profondeur
        self.min_echantillons = min_echantillons
        self.racine = None
    
    def entropie(self, y):
        """Calcule l'entropie d'un ensemble"""
        compteur = Counter(y)
        entropie = 0
        for count in compteur.values():
            p = count / len(y)
            if p > 0:
                entropie -= p * np.log2(p)
        return entropie
    
    def gain_information(self, X_colonne, y, seuil):
        """Calcule le gain d'information pour une division"""
        # Division
        masque_gauche = X_colonne <= seuil
        masque_droite = ~masque_gauche
        
        if sum(masque_gauche) == 0 or sum(masque_droite) == 0:
            return 0
        
        # Entropie avant division
        entropie_parent = self.entropie(y)
        
        # Entropie après division
        n = len(y)
        n_gauche, n_droite = sum(masque_gauche), sum(masque_droite)
        e_gauche = self.entropie(y[masque_gauche])
        e_droite = self.entropie(y[masque_droite])
        entropie_enfants = (n_gauche/n) * e_gauche + (n_droite/n) * e_droite
        
        return entropie_parent - entropie_enfants
    
    def meilleure_division(self, X, y):
        """Trouve la meilleure division possible"""
        meilleur_gain = -1
        meilleur_feature = None
        meilleur_seuil = None
        
        for feature_idx in range(X.shape[1]):
            valeurs = np.unique(X[:, feature_idx])
            
            for seuil in valeurs:
                gain = self.gain_information(X[:, feature_idx], y, seuil)
                
                if gain > meilleur_gain:
                    meilleur_gain = gain
                    meilleur_feature = feature_idx
                    meilleur_seuil = seuil
        
        return meilleur_feature, meilleur_seuil
    
    def construire_arbre(self, X, y, profondeur=0):
        """Construit l'arbre récursivement"""
        n_echantillons, n_features = X.shape
        n_classes = len(np.unique(y))
        
        # Conditions d'arrêt
        if (profondeur >= self.max_profondeur or 
            n_classes == 1 or 
            n_echantillons < self.min_echantillons):
            valeur_feuille = Counter(y).most_common(1)[0][0]
            return NoeudDecision(valeur=valeur_feuille)
        
        # Trouver la meilleure division
        meilleur_feature, meilleur_seuil = self.meilleure_division(X, y)
        
        if meilleur_feature is None:
            valeur_feuille = Counter(y).most_common(1)[0][0]
            return NoeudDecision(valeur=valeur_feuille)
        
        # Diviser les données
        masque_gauche = X[:, meilleur_feature] <= meilleur_seuil
        masque_droite = ~masque_gauche
        
        # Construire récursivement
        gauche = self.construire_arbre(X[masque_gauche], y[masque_gauche], profondeur + 1)
        droite = self.construire_arbre(X[masque_droite], y[masque_droite], profondeur + 1)
        
        return NoeudDecision(meilleur_feature, meilleur_seuil, gauche, droite)
    
    def fit(self, X, y):
        """Entraîne l'arbre"""
        self.racine = self.construire_arbre(X, y)
    
    def predire_un(self, x, noeud):
        """Prédit la classe d'un échantillon"""
        if noeud.valeur is not None:
            return noeud.valeur
        
        if x[noeud.feature] <= noeud.seuil:
            return self.predire_un(x, noeud.gauche)
        return self.predire_un(x, noeud.droite)
    
    def predict(self, X):
        """Prédit les classes de plusieurs échantillons"""
        return np.array([self.predire_un(x, self.racine) for x in X])


# Utilisation
if __name__ == "__main__":
    # Données d'exemple (Iris simplifié)
    from sklearn.datasets import load_iris
    from sklearn.model_selection import train_test_split
    
    iris = load_iris()
    X_train, X_test, y_train, y_test = train_test_split(
        iris.data, iris.target, test_size=0.2, random_state=42
    )
    
    # Entraîner notre arbre
    arbre = ArbreDecision(max_profondeur=5)
    arbre.fit(X_train, y_train)
    
    # Prédictions
    predictions = arbre.predict(X_test)
    
    # Précision
    precision = np.mean(predictions == y_test)
    print(f"Précision : {precision:.2%}")

7. Élagage d'Arbres (Pruning)

⚠️ Problème du surapprentissage

Un arbre trop profond mémorise les données d'entraînement au lieu d'apprendre les patterns généraux → mauvaises performances sur nouvelles données.

Techniques d'élagage

1. Pré-élagage (Pre-pruning)

Arrêter la croissance de l'arbre avant qu'il ne soit complet :

  • max_depth : profondeur maximale
  • min_samples_split : minimum d'échantillons pour diviser
  • min_samples_leaf : minimum d'échantillons par feuille
  • max_leaf_nodes : nombre maximal de feuilles

2. Post-élagage (Post-pruning)

Construire l'arbre complet puis supprimer les branches qui n'améliorent pas la performance :

  • Reduced Error Pruning : supprimer les nœuds qui ne dégradent pas la validation
  • Cost Complexity Pruning : équilibrer complexité et erreur
# Exemple de pré-élagage avec sklearn
from sklearn.tree import DecisionTreeClassifier

clf = DecisionTreeClassifier(
    max_depth=5,              # Profondeur max : 5
    min_samples_split=10,     # Min 10 échantillons pour diviser
    min_samples_leaf=5,       # Min 5 échantillons par feuille
    max_leaf_nodes=20         # Max 20 feuilles
)

clf.fit(X_train, y_train)

8. Points Clés à Retenir

✅ Résumé

  • Les arbres de décision sont des modèles supervisés pour classification et régression
  • Construction basée sur le gain d'information (entropie) ou indice de Gini
  • Très interprétables mais risque de surapprentissage
  • L'élagage est essentiel pour de bonnes performances
  • Base des forêts aléatoires (Random Forests) et du boosting

🔑 Formules essentielles

Entropie : H(S) = -Σ pᵢ·log₂(pᵢ)
Gain : Gain(S,A) = H(S) - Σ (|Sᵥ|/|S|)·H(Sᵥ)
Gini : Gini(S) = 1 - Σ pᵢ²
