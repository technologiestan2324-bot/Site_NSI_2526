Parcours Infixe avec pile

Ce que doit faire le programme

Voici un schéma illustrant l'évolution de la pile dans l'exemple du parcours infixe (gauche → racine → droite) pour l'arbre suivant :

markdown
CopierModifier
        1
       / \
      2   3
     / \
    4   5

Les étapes décrivent le contenu de la pile et la valeur du nœud current à chaque itération.

Départ

  • Pile : vide
  • Current : 1 (racine) (que voue pouvez appeler pointeur aussi)

Étape 1 : Descente à gauche

  • On empile 1 → 2 → 4 (descente à gauche complète).
  • Pile : [1, 2, 4]
  • Current : None (plus de fils gauche pour 4)

Étape 2 : Traiter le nœud 4

  • Dépiler 4, afficher 4, puis passer à son fils droit (None).
  • Pile : [1, 2]
  • Current : None
  • Sortie : 4

Étape 3 : Remonter au nœud 2

  • Dépiler 2, afficher 2, puis passer à son fils droit (5).
  • Pile : [1]
  • Current : 5
  • Sortie : 4, 2

Étape 4 : Descente sur 5

  • Empiler 5 (descente à gauche complète).
  • Pile : [1, 5]
  • Current : None (5 n'a pas de fils gauche)

Étape 5 : Traiter le nœud 5

  • Dépiler 5, afficher 5, puis passer à son fils droit (None).
  • Pile : [1]
  • Current : None
  • Sortie : 4, 2, 5

Étape 6 : Remonter au nœud 1

  • Dépiler 1, afficher 1, puis passer à son fils droit (3).
  • Pile : []
  • Current : 3
  • Sortie : 4, 2, 5, 1

Étape 7 : Descente sur 3

  • Empiler 3 (descente à gauche complète).
  • Pile : [3]
  • Current : None (3 n'a pas de fils gauche)

Étape 8 : Traiter le nœud 3

  • Dépiler 3, afficher 3, puis passer à son fils droit (None).
  • Pile : []
  • Current : None
  • Sortie : 4, 2, 5, 1, 3
