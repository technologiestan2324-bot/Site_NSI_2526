Parcours préfixe avec pile

Le parcours préfixé avec une pile simule la récursivité du parcours préfixé traditionnel en utilisant explicitement une structure de pile (stack). Cette méthode est utile pour comprendre comment les appels récursifs fonctionnent en arrière-plan.

Algorithme pas à pas

L'idée est la suivante :

  1. Initialiser une pile avec la racine de l'arbre.
  1. Tant que la pile n'est pas vide :
    • Retirer le nœud en haut de la pile.
    • Traiter (afficher) ce nœud.
    • Ajouter le sous-arbre droit à la pile (si non nul).
    • Ajouter le sous-arbre gauche à la pile (si non nul).
  1. Répéter jusqu'à ce que la pile soit vide.

On ajoute d'abord le sous-arbre droit puis le sous-arbre gauche pour que le sous-arbre gauche soit traité en premier (puisque la pile est LIFO - Last In, First Out).

Algorithme (pseudocode)

fonction parcoursPréfixeAvecPile(racine):
    si racine est nul:
        retourner

    pile ← pile vide
    empiler(racine, pile)

    tant que pile n'est pas vide:
        nœud ← dépiler(pile)         // Étape 1 : Retirer un nœud
        afficher(nœud.valeur)        // Étape 2 : Traiter le nœud
        si nœud.droit ≠ nul:
            empiler(nœud.droit, pile)  // Étape 3 : Ajouter le sous-arbre droit
        si nœud.gauche ≠ nul:
            empiler(nœud.gauche, pile) // Étape 4 : Ajouter le sous-arbre gauche

Exemple avec un arbre

Prenons cet arbre binaire :

mathematica
CopierModifier
       A
      / \
     B   C
    / \
   D   E

Étapes du parcours préfixé avec une pile :

  1. Initialisation : pile = [A]
  1. Dépiler A, traiter : résultat = [A], empiler C, puis B → pile = [C, B]
  1. Dépiler B, traiter : résultat = [A, B], empiler E, puis D → pile = [C, E, D]
  1. Dépiler D, traiter : résultat = [A, B, D] → pile = [C, E]
  1. Dépiler E, traiter : résultat = [A, B, D, E] → pile = [C]
  1. Dépiler C, traiter : résultat = [A, B, D, E, C] → pile = []

Résultat final : A, B, D, E, C.

Explication étape par étape


Étape 1 : Comprendre la structure de l'arbre

Un arbre binaire est composé de nœuds. Chaque nœud contient :

  • Une valeur (donnée stockée dans le nœud).
  • Un fils gauche (sous-arbre gauche).
  • Un fils droit (sous-arbre droit).

On doit d'abord définir la structure d'un nœud.

Étape 2 : Initialiser la pile

Le parcours préfixé commence par la racine. On utilise une pile (une liste en Python) pour gérer les nœuds à visiter.

  • Étape 3.1 : Initialiser la pile avec la racine.
  • Étape 3.2 : Tant que la pile n'est pas vide, continuer le traitement.

Étape 4 : Parcourir les nœuds

Pour chaque nœud retiré de la pile :

  1. Traiter le nœud courant (afficher sa valeur).
  1. Ajouter d'abord le fils droit, puis le fils gauche à la pile (si ces fils existent).

Cela garantit que le fils gauche est traité avant le fils droit, car la pile fonctionne en LIFO (Last In, First Out).
